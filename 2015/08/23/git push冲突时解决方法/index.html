git push冲突时解决方法

在使用git中多人同时修改一个文件时，比如A修改test文件并push到远程仓库，B也修改test文件并push到远程，这时Bgit push会提示相应的冲突错误。

解决方法：

  • 第一步：从项目仓库中，创建一个新的分支，并测试合并请求中的改动
  • 第二步: 创建本地分支local用来存放远程仓库的最新分支
  • 第三步：进行合并，手动修改合并后的文件
  • 第四步：修改后按照正常的git流程进行操作
